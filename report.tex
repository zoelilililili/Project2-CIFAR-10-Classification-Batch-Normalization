\section{Building a Custom Network for CIFAR-10}

\subsection{Network Architecture}

We design a ResNet-style convolutional neural network named \texttt{MyCIFARNet} for CIFAR-10 classification. The architecture consists of an initial convolutional layer followed by four stages of residual blocks, finishing with an adaptive average pooling layer and a classifier head.

\subsubsection{Architecture Overview}

\begin{figure}[H]
\centering
\includegraphics[width=\textwidth]{figures/architecture.png}
\caption{Architecture of MyCIFARNet.}
\label{fig:arch}
\end{figure}

The network processes $32\times32\times3$ CIFAR-10 images through the following stages:

\begin{enumerate}
    \item \textbf{Initial Conv}: $3\times3$ convolution with $3 \rightarrow \text{base\_ch}$ channels, BatchNorm, ReLU.
    \item \textbf{Stage 1}: $2\times$ ResidualBlock (base\_ch $\rightarrow$ base\_ch), spatial size $32\times32$.
    \item \textbf{Stage 2}: ResidualBlock (base\_ch $\rightarrow 2\times\text{base\_ch}$, stride=2) + ResidualBlock ($2\times\text{base\_ch} \rightarrow 2\times\text{base\_ch}$), spatial $16\times16$.
    \item \textbf{Stage 3}: ResidualBlock ($2\times\text{base\_ch} \rightarrow 4\times\text{base\_ch}$, stride=2) + ResidualBlock ($4\times\text{base\_ch} \rightarrow 4\times\text{base\_ch}$), spatial $8\times8$.
    \item \textbf{Stage 4}: ResidualBlock ($4\times\text{base\_ch} \rightarrow 8\times\text{base\_ch}$, stride=2) + ResidualBlock ($8\times\text{base\_ch} \rightarrow 8\times\text{base\_ch}$), spatial $4\times4$.
    \item \textbf{Pooling}: AdaptiveAvgPool2d(1) $\rightarrow$ Flatten.
    \item \textbf{Classifier}: FC($8\times\text{base\_ch} \rightarrow 256$) $\rightarrow$ ReLU $\rightarrow$ Dropout $\rightarrow$ FC($256 \rightarrow 10$).
\end{enumerate}

\subsubsection{Residual Block}

Each ResidualBlock implements the standard residual connection:
\begin{equation}
    \text{output} = \text{ReLU}(\text{BN}(\text{Conv}_{3\times3}(\text{BN}(\text{Conv}_{3\times3}(x)))) + \text{skip}(x))
\end{equation}
When dimensions change (stride $\neq$ 1 or channel mismatch), the skip connection uses a $1\times1$ convolution with BatchNorm.

\subsubsection{Implementation}

\begin{lstlisting}[caption={ResidualBlock and MyCIFARNet implementation}, label={lst:net}]
class ResBlock(nn.Module):
    def __init__(self, in_ch, out_ch, stride=1, activation='relu'):
        super().__init__()
        act_fn = nn.ReLU() if activation == 'relu' else nn.LeakyReLU(0.1)
        self.conv1 = nn.Conv2d(in_ch, out_ch, 3, stride, 1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_ch)
        self.act1 = act_fn
        self.conv2 = nn.Conv2d(out_ch, out_ch, 3, 1, 1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_ch)
        self.skip = nn.Identity()
        if stride != 1 or in_ch != out_ch:
            self.skip = nn.Sequential(
                nn.Conv2d(in_ch, out_ch, 1, stride, bias=False),
                nn.BatchNorm2d(out_ch))
        self.act2 = act_fn

    def forward(self, x):
        identity = self.skip(x)
        out = self.act1(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        return self.act2(out + identity)

class MyCIFARNet(nn.Module):
    def __init__(self, base_ch=64, num_classes=10,
                 activation='relu', dropout=0.3):
        super().__init__()
        self.conv1 = nn.Conv2d(3, base_ch, 3, 1, 1, bias=False)
        self.bn1 = nn.BatchNorm2d(base_ch)

        stages_config = [
            (base_ch,      base_ch,     2, 1),
            (base_ch,      base_ch*2,   2, 2),
            (base_ch*2,    base_ch*4,   2, 2),
            (base_ch*4,    base_ch*8,   2, 2),
        ]
        self.stages = nn.ModuleList()
        for in_ch, out_ch, n_blocks, stride in stages_config:
            stage = [ResBlock(in_ch, out_ch, stride)]
            for _ in range(n_blocks - 1):
                stage.append(ResBlock(out_ch, out_ch, 1))
            self.stages.append(nn.Sequential(*stage))

        self.pool = nn.AdaptiveAvgPool2d(1)
        self.classifier = nn.Sequential(
            nn.Flatten(), nn.Linear(base_ch*8, 256), nn.ReLU(),
            nn.Dropout(dropout), nn.Linear(256, num_classes))

    def forward(self, x):
        out = F.relu(self.bn1(self.conv1(x)))
        for stage in self.stages:
            out = stage(out)
        out = self.pool(out)
        return self.classifier(out)
\end{lstlisting}

\subsubsection{Design Rationale}

The network incorporates all required and optional components specified in the project:
\begin{itemize}
    \item \textbf{Fully-Connected layer}: 2 Linear layers in classifier.
    \item \textbf{2D Convolutional layer}: 17 Conv2d layers (2 per ResBlock $\times$ 8 blocks + 1 initial).
    \item \textbf{2D Pooling layer}: AdaptiveAvgPool2d at the network bottleneck.
    \item \textbf{Activation functions}: ReLU / LeakyReLU throughout.
    \item \textbf{Batch Normalization}: Applied after every Conv2d.
    \item \textbf{Dropout}: Applied between FC layers (rate = 0.3).
    \item \textbf{Residual Connection}: Skip connections in every ResBlock.
\end{itemize}

Three variants are provided by varying \texttt{base\_ch}: S (32), M (48), and L (64), with model sizes of approximately 2.9M, 6.5M, and 11.3M parameters respectively.

\subsection{Discussion}

The ablation experiments reveal several key insights about the behavior of our network:

\textbf{Filter Count (E1):} Increasing model capacity from S (2.9M) to M (6.5M) yields a clear improvement from 83.09\% to 84.14\%. However, further scaling to L (11.3M) provides negligible additional gain (84.12\%), suggesting diminishing returns from raw parameter increase. This indicates that the M variant already has sufficient representational capacity for CIFAR-10, and the additional 4.8M parameters in the L variant are not effectively utilized under the current training setup (30 epochs, Adam, lr=$10^{-3}$).

\textbf{Activation Function (E2):} ReLU outperforms LeakyReLU by 0.67 percentage points (84.14\% vs. 83.47\%). While LeakyReLU is designed to address the ``dying ReLU'' problem by allowing small negative gradients, this advantage appears unnecessary in our network where Batch Normalization already ensures healthy gradient flow throughout training.

\textbf{L2 Regularization (E3):} This experiment produced the most striking result. Removing L2 regularization entirely (weight\_decay=0) increased accuracy from 84.14\% to 84.96\%, a substantial 0.82 percentage point improvement. Stronger regularization ($5\times10^{-3}$) degraded performance severely to 81.45\%. This counter-intuitive outcome suggests that our network, equipped with Batch Normalization and Dropout, already has sufficient implicit regularization. Adding explicit L2 penalty unnecessarily constrains the model, preventing it from fitting the training data effectively. This aligns with recent findings that BN itself acts as a regularizer \cite{bn_neurips2018}, making additional weight decay redundant or even harmful for this architecture.

\textbf{Optimizer Choice (E4):} Adam (84.14\%) outperforms both SGD with momentum (82.84\%) and RMSprop (82.52\%) by a significant margin of approximately 1.3--1.6 percentage points. Adam's adaptive per-parameter learning rates are particularly beneficial for networks with Batch Normalization, as they automatically adjust to the varying gradient scales introduced by the normalization layers.

\textbf{Overall Architecture Assessment:} MyCIFARNet achieves a best test accuracy of 84.96\% (error rate 15.04\%), which is competitive with VGG-A+BN (83.36\%) despite having a completely different structural design. The ResNet-style skip connections combined with deep stages (up to 8$\times$ base\_ch channels) provide effective gradient propagation and feature learning. The network's main strength lies in its balanced combination of residual learning, Batch Normalization, and moderate dropout regularization. However, the accuracy plateau between the M and L variants suggests that further improvements may require longer training schedules, learning rate decay, or data augmentation rather than simply increasing model size.

\subsection{How Does Batch Normalization Help Optimization?}

To understand the mechanisms behind BN's effectiveness, we analyze three aspects of the optimization landscape based on \cite{bn_neurips2018}.

\subsubsection{Loss Landscape (Lipschitzness of the Loss)}

To test the impact of BN on the stability of the loss function, we measure the loss variation across models trained with different learning rates. At each epoch, we record the training loss for all four models and compute the minimum and maximum values, which together form the ``loss landscape'' envelope.

\begin{figure}[H]
\centering
\begin{subfigure}{0.48\textwidth}
    \centering
    \includegraphics[width=\textwidth]{figures/loss_landscape.png}
    \caption{Loss landscape for VGG-A (left) and VGG-A+BN (right).}
\end{subfigure}
\hfill
\begin{subfigure}{0.48\textwidth}
    \centering
    \includegraphics[width=\textwidth]{figures/loss_landscape_comparison.png}
    \caption{Overlay comparison on the same axes.}
\end{subfigure}
\caption{Loss landscape analysis. A wider spread (larger area between max and min curves) indicates higher sensitivity to the learning rate, i.e., a less smooth loss landscape. BN typically reduces this spread.}
\label{fig:landscape}
\end{figure}

The fill between the min and max loss curves represents the variation of loss values across different step sizes. A smaller filled area indicates a smoother loss landscape --- meaning the loss is less sensitive to the choice of learning rate. We expect BN to significantly reduce this variation, confirming that it makes the optimization landscape more Lipschitz-smooth.

\subsubsection{Gradient Predictiveness}

Gradient predictiveness measures how well the local linear approximation (first-order Taylor expansion) of the loss predicts the actual change in loss along the negative gradient direction. Formally, for a given parameters $\theta$ and gradient $g = \nabla L(\theta)$, we compare:

\begin{equation}
    \text{Predicted: } L(\theta - \alpha g) \approx L(\theta) - \alpha \|g\|^2
\end{equation}

against the actual loss $L(\theta - \alpha g)$ for various step sizes $\alpha$.

\begin{figure}[H]
\centering
\includegraphics[width=\textwidth]{figures/gradient_predictiveness.png}
\caption{Gradient predictiveness: actual loss vs Taylor approximation for VGG-A (No BN, left) and VGG-A+BN (right). The dashed line is the first-order Taylor prediction; the solid line is the actual loss.}
\label{fig:gp}
\end{figure}

\begin{figure}[H]
\centering
\includegraphics[width=0.7\textwidth]{figures/gradient_predictiveness_error.png}
\caption{Prediction error $|\text{Actual} - \text{Predicted}|$ for BN vs No BN. A smaller error indicates that the first-order gradient information is more reliable for predicting loss changes, implying a smoother optimization landscape.}
\label{fig:gp_err}
\end{figure}

Models with BN typically exhibit smaller prediction error, meaning the gradient direction provides a more reliable estimate of how the loss will change --- an indication of a smoother loss surface.

\subsubsection{Gradient Smoothness (Maximum Difference in Gradient)}

We measure the local Lipschitz constant of the gradient by evaluating:

\begin{equation}
    \frac{\|g(\theta + \delta) - g(\theta)\|}{\|\delta\|}
\end{equation}

for small random perturbations $\delta$ at various scales $\varepsilon$. A smaller value indicates a smoother gradient field, which is beneficial for first-order optimization methods.

\begin{figure}[H]
\centering
\includegraphics[width=0.7\textwidth]{figures/gradient_smoothness.png}
\caption{Gradient smoothness comparison across different perturbation scales $\varepsilon$. Lower values indicate smoother gradients.}
\label{fig:gs}
\end{figure}

\subsection{Discussion on Batch Normalization}

The experimental results provide compelling evidence that Batch Normalization significantly improves the training of VGG-A on CIFAR-10 across multiple dimensions.

\textbf{Training Performance:} Across all four learning rates, VGG-A with BN consistently outperforms the baseline VGG-A without BN. The best BN model achieves 83.36\% (lr=$10^{-3}$) compared to only 78.69\% for the best No-BN model (lr=$5\times10^{-4}$), representing an improvement of 4.67 percentage points. More importantly, BN makes the network substantially more robust to the choice of learning rate: the BN accuracy range across LRs is only 75.06--83.36\% (spread of 8.30\%), whereas the No-BN model fluctuates between 74.29--78.69\% (spread of 4.40\% but at a consistently lower level). Notably, at the lowest learning rate ($10^{-4}$), BN underperforms No-BN (75.06\% vs. 77.00\%), suggesting that the regularizing effect of BN may require sufficiently large learning rates to be beneficial.

\textbf{Loss Landscape Analysis:} The loss landscape visualization (Figure~\ref{fig:landscape}) demonstrates that BN reduces the sensitivity of the optimization trajectory to the learning rate. The narrower min-max envelope of VGG-A+BN indicates a smoother loss surface where gradient steps of different sizes lead to more consistent loss reduction. This Lipschitz smoothness property explains why BN-trained models converge more reliably across different hyperparameter settings.

\textbf{Gradient Predictiveness:} As shown in Figure~\ref{fig:gp} and~\ref{fig:gp_err}, the first-order Taylor approximation is significantly more accurate for BN models than for their No-BN counterparts. This means that with BN, the local gradient provides a more reliable estimate of how the loss will change, enabling the optimizer to take more effective steps. The smaller prediction error directly contributes to faster and more stable convergence.

\textbf{Gradient Smoothness:} The gradient smoothness measurement (Figure~\ref{fig:gs}) quantifies that BN reduces the local Lipschitz constant of the gradient. Under small parameter perturbations, the BN model's gradient varies less than the No-BN model's gradient. This property is particularly important for first-order optimization methods, as it ensures that the gradient direction remains consistent within a local neighborhood of the current parameters.

\textbf{Conclusion:} Collectively, these four perspectives---raw accuracy improvement, loss landscape Lipschitzness, gradient predictiveness, and gradient smoothness---paint a consistent picture: Batch Normalization fundamentally reparameterizes the optimization problem to make it more well-conditioned. Rather than merely reducing internal covariate shift (the original motivation), BN achieves its effectiveness by smoothing the optimization landscape, making gradient-based training faster, more stable, and more robust to hyperparameter choices. This conclusion aligns with the theoretical analysis presented by Santurkar et al.~\cite{bn_neurips2018}, and our experiments provide empirical validation on the CIFAR-10 dataset using the VGG-A architecture.
